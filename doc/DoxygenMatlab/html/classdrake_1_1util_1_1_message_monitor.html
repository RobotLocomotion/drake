Detailed Description

Monitors a particular lcm message type (specified in the constructor) and maintains the most up-to-date instance of that type.

The lcm message type must have a .timestamp field. % If messages arrive out of order, it will keep the message with the largest timestamp. If no message is received for timeout seconds then the timestamp history is reset (e.g, it's assumed that the channel went dead and so accepts a time starting back at zero). The default value of this timeout is 1 second.

Note that this is implemented with reflection instead of generics (e.g. MessageMonitor<lcmtype>) because the binding needs to happen at run-time (e.g. from MATLAB).
