## Build system

<!-- <relnotes for cmake,doc,setup,third_party,tools go here> -->

* [fix] TBD [tools,cmake] [cmake] Guard against lld multi-threading ([#24457][_#24457])  # If the user ends up linking Drake with lld, add --threads=1 to the linker command to prevent overloading the machine.
* [fix] TBD [tools,setup] [setup] Add lld as a build-dep on Ubuntu Noble ([#24442][_#24442])  # Without lld installed, rules_cc defaults to using ld.gold, which is deprecated and produces warnings when linking Rust code. With lld installed, rules_cc uses it (no matter if the selected compiler is GCC or Clang). Add --threads=2 linkopts to prevent lld from overloading the machine. Luckily, mold also supports this flag with the same meaning, so we can drop the mold-specific --thread-count=2 flag.
* [fix] TBD [setup,workspace,geometry,tools,doc] [setup] Consolidate prereqs scripts ([#24458][_#24458])  # Tracing through multiple files is needlessly confusing, especially on Ubuntu where the requirement or (anti-requirement) of 'sudo' varies, and ends up with a lot of copy-paste across the files. Fold all bash scripts down to a single file, except for the binary prereqs script, which gets installed separately. On Ubuntu, rework the logic to uniformly run as non-root except for ${maybe_sudo} on specific command that need elevated privileges. Release note: install_prereqs should no longer be run under sudo.
* [fix] TBD [cmake,tools] Update dependency rules_cc to v0.2.18 ([#24470][_#24470])
* Remove unused / unsupported install_prereqs flags ([#24388][_#24388])

# Notes


This release provides
[pre-compiled binaries](https://github.com/RobotLocomotion/drake/releases/tag/v1.53.0).
See [Installation](/installation.html) for instructions on how to use them.

Drake binary releases incorporate a pre-compiled version of
[SNOPT](https://ccom.ucsd.edu/~optimizers/solvers/snopt/) as part of the
[Mathematical Program toolbox](/doxygen_cxx/group__solvers.html). Thanks to
Philip E. Gill and Elizabeth Wong for their kind support.
